# PRD 智能任务拆解器

## 功能描述

将中文产品需求文档（PRD）自动拆解为开发任务清单，输出工时估算、依赖关系、风险识别和 PRD 质量体检报告。

## 使用场景

当用户说以下类似的话时，调用此 Skill：
- "帮我拆解这份 PRD"
- "分析一下这个需求文档"
- "把这个 PRD 转成任务清单"
- "估算一下这个需求的工时"
- "帮我检查这份 PRD 有没有问题"

## 调用方式

```bash
# 生成 Markdown 报告（默认）
python main.py <prd文件路径>

# 生成 CSV（可导入飞书/钉钉）
python main.py <prd文件路径> --format csv

# 生成 JSON（接入自动化工具链）
python main.py <prd文件路径> --format json

# 指定输出文件名
python main.py <prd文件路径> --output <输出文件名>
```

## 输出内容

- 识别功能模块数量
- 拆解开发任务（含 ID、类型、优先级、工时估算、依赖关系、验收标准、风险提示）
- 总工时和预计工期
- PRD 质量问题清单（模糊词汇、缺少验收标准、缺少异常流程等）

## 支持的 PRD 格式

- Markdown 标题（`## 模块名`）
- 中文序号（`一、二、三`）
- 数字序号（`1. 2. 3.`）
- 【】格式（`【模块名】`）
- 普通 `.txt` 文本

## 示例

用户输入：
> 帮我拆解 user_center_prd.md 这份需求文档，输出 CSV 格式

执行命令：
```bash
python main.py user_center_prd.md --format csv
```

## 注意事项

- 需要 Python 3.8 或以上
- 零外部依赖，无需安装任何第三方库
- PRD 文件需为 UTF-8 编码
- 文件内容少于 50 字符时会提示内容过短
